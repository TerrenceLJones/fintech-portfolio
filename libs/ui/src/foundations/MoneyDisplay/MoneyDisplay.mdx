# MoneyDisplay

Formats a currency amount for display, with loading/credit/debit states and an optional
"derived" mode for values that are ledger projections rather than editable inputs.

## User story

**US-CW-023 — Component library — atoms & foundations**

As a clearline-web engineer, I want StatusBadge/MoneyDisplay/BudgetGauge and the 8 Tier-1 atoms
available as typed React components, so that I never hand-roll a button, badge, or money value
and risk missing an accessibility or formatting rule the design system already solved.

## Acceptance criteria

- AC-01 — StatusBadge always shows an icon and text label alongside its color.
- **AC-02 — MoneyDisplay.** Given MoneyDisplay with `state="loading"`, when it renders, then it
  shows a skeleton sized to the final glyphs, never a flash of "$0.00".
- **AC-03 — MoneyDisplay.** Given MoneyDisplay with `derived={true}`, when it renders, then it
  shows a "DERIVED · READ-ONLY" chip instead of any editable-looking affordance.
- AC-04 — BudgetGauge spells out the percentage and any overage amount in text.
- AC-05 — AC-07 cover the remaining Tier-1 atoms (Checkbox, Button, TextField) in this same
  story; see their own docs pages for the full text.

### Edge case covered here

Credit/debit states must show a directional icon (up/down arrow) in addition to color, so the
direction of money movement is never conveyed by color alone.

## Accessibility (EPIC-CW-014 · US-CW-020 AC-03)

The visual glyphs (e.g. `$1,999.00`) are marked `aria-hidden`, and the amount is exposed to
screen readers as a fully spoken, visually-hidden phrase — "One thousand nine hundred ninety-nine
dollars" — via `spokenMoneyAmount` from `@clearline/money`. Credit/debit direction, otherwise
carried only by an arrow icon and color, is spoken too ("Credit of …"). Pass `currency` so the
spoken unit matches the glyphs (e.g. "pounds"), or `ariaLabel` to override the phrase entirely.
