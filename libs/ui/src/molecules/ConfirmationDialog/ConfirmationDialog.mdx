# ConfirmationDialog

A Radix-`Dialog`-backed confirmation for irreversible actions, with a countdown that keeps the
confirm button disabled for a beat before arming it.

## User story

**US-CW-024 — Component library — molecules**

As a clearline-web engineer, I want Modal/ConfirmationDialog and the other 6 Tier-2 molecules
available as typed React components built on Radix Dialog, so that every irreversible-action
confirmation in the app gets the same focus-trapped, Escape-cancels, countdown-gated behavior for
free, instead of each feature epic re-implementing dialog accessibility by hand.

## Acceptance criteria

- AC-01 — Modal exposes `role="dialog"`, traps focus, and closes via Escape/Cancel.
- **AC-02 — ConfirmationDialog.** Given a ConfirmationDialog with `countdown={3}`, when it opens,
  then the confirm button is gated (via `aria-disabled`, not the native `disabled` attribute, so
  it stays focusable) and reads "Confirm in 3…", with an `aria-describedby`-linked, visually-hidden
  explanation of the wait, counting down each second until armed, and re-gates on every subsequent
  open transition even if previously armed.
- AC-03 — VirtualCard hides the derived remaining-limit figure entirely when frozen.
- AC-04 — Timeline shows before → after diffs with an arrow between them.
- AC-05 — AIInsightCard's primary/secondary actions independently invoke their own callbacks.

### Edge case covered here

`countdown={0}` is immediately armed — no artificial wait when the caller opts out of the delay.
