# RevealSecretModal

The reveal-once secret modal (design §19.3, reused for §19.5 webhook signing secrets). A newly minted
secret is shown in full **exactly once**: a copy button, a can't-miss warning that it won't be shown
again, and a single acknowledge-and-close action.

## User story

**US-CW-041 — Developer Settings — API Keys & Webhooks**

As an Admin or Owner, I want the full API key / webhook signing secret shown once at creation and never
re-displayable, so that secrets behave like production fintech developer settings.

## Acceptance criteria

- **AC-01 / AC-06 — Reveal once.** On creation the full secret is shown with a copy button and a bold
  warning ("This is the only time you'll see this key…"). After dismissal only the masked form remains.
- **AC-02 — No re-reveal.** There is deliberately no re-open/re-reveal path — a lost secret is
  recovered only by revoke + recreate.

### Notes

- Presentational: the caller owns when it opens and what `onDone` does (typically invalidating the
  list so the masked form appears).
- Reused for both API keys and webhook signing secrets — only the copy strings differ.
