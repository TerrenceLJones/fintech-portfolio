# ApprovalActionBar

Approve/Reject/Escalate actions that disable Approve with a stated reason (RBAC /
separation-of-duties) rather than silently omitting it.

## User story

**US-CW-025 — Component library — organisms & decorators**

As a clearline-web engineer, I want DataTable/ApprovalActionBar/BulkActionResult/EmptyState and
the AppShell decorators available as typed React components, so that I
can assemble a full screen (selection, sorting, approval flows, partial-failure summaries) by
composing already-built pieces instead of writing table selection/sort logic or approval-limit
disabling logic from scratch in each feature epic.

## Acceptance criteria

- AC-01 — AC-02 — DataTable's select-all toggle and vendor-column sort.
- **AC-03 — ApprovalActionBar.** Given ApprovalActionBar with a `reason` prop set (e.g.
  separation-of-duties) and no explicit `canApprove`, when it renders, then Approve is gated (via
  Button's `aria-disabled`, staying focusable) and the reason is shown in text beneath the
  buttons, programmatically linked to Approve via `aria-describedby` — never silently missing,
  and never only visually adjacent.
- AC-04 — BulkActionResult excludes failures from the committed count and lists each reason.
- AC-05 — AppShell owns theme state and requires zero additional per-page token setup.

### Edge case covered here

The Escalate action can be hidden (`showEscalate={false}`) independently of whether
approve/reject are disabled.
