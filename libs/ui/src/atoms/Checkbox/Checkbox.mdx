# Checkbox

Wraps Radix `Checkbox` (rather than a hand-rolled implementation) for focus/keyboard/ARIA
handling, with checked/unchecked/indeterminate states.

## User story

**US-CW-023 — Component library — atoms & foundations**

As a clearline-web engineer, I want StatusBadge/MoneyDisplay/BudgetGauge and the 8 Tier-1 atoms
available as typed React components, so that I never hand-roll a button, badge, or money value
and risk missing an accessibility or formatting rule the design system already solved.

## Acceptance criteria

- AC-01 — AC-04 — StatusBadge/MoneyDisplay/BudgetGauge, covered on their own docs pages.
- **AC-05 — Checkbox.** Given Checkbox, which wraps Radix `Checkbox` rather than a hand-rolled
  implementation, when a user clicks it or activates it via keyboard (Space), then
  `onCheckedChange` fires and `aria-checked` reflects checked/indeterminate/unchecked correctly.
- AC-06 — Button's `loading` state disables the button and blocks `onClick`.
- AC-07 — TextField's `state="error"` sets `aria-invalid` and `aria-describedby`.

### Edge case covered here

The indeterminate state is exercised as its own story — it must be visually and semantically
distinct from both checked and unchecked, not a visual approximation of either.
